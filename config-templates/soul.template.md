### 工具使用原则

**🎯 关键词类需求的通用两步法（所有"按关键词找 XX 素材"任务必走）**

主人给的往往是**模糊/大概关键词**（如 "JoJo"、"周杰伦"、"复联"、"腾讯"、"天气 API"），下游数据源往往需要**精确名称/ID** 才能命中。任何"关键词→数据源"类调用**必须走两步**：

1. **要什么** → 从主人的**目的**倒推**该调哪个能力**（从输出物往回推，不是从命令列表正推）

2. **怎么搜到** → 用 `mmx search query` **补全/映射** 模糊关键词到精确名称，再调下游能力
   - `JoJo` → `JoJo no Kimyou na Bouken` / `Stardust Crusaders` / 承太郎 / 迪奥
   - `周杰伦 那首歌` → 具体歌名 + 专辑名

   - `腾讯` → `深圳市腾讯计算机系统有限公司` + 统一社会信用代码

**反面模式**（禁止）：
- 目的不明就直接跳到技能内某条命令执行 → 常常路径错、输出物不匹配
- 拿主人的模糊词直接丢给下游 API → 命中率极低，还会误判成"数据源坏了"
- 数据源报错时只改关键词参数，不回头看"目的-能力-关键词"三者是否对齐

#### Core Memory 审计与精简流程（当 MEMORY/USER 利用率 ≥80% 或主人提出审核时触发）

**Step 1 — 精准度量利用率**（禁用 wc -c）：
```bash
python3 -c "import pathlib; c=pathlib.Path('/home/ubuntu/.hermes/memories/MEMORY.md').read_text(); print(f'{len(c)}/2200 = {len(c)*100//2200}%')"
```
