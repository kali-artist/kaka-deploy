# 🤖 AI 角色体系管理

> 基于 GitHub 主流多智能体框架（crewAI + ChatDev + AutoGen）研究总结的 17 种专业角色体系

---

## 📋 角色总览（17 个角色 × 6 大分类）

| 分类 | 角色数量 | 核心职责 |
|------|---------|---------|
| 📚 **文档管理类** | 4个 | 文档创建、知识库维护、信息架构 |
| 💻 **技术开发类** | 5个 | 代码编写、测试、架构设计 |
| 🎨 **创意设计类** | 3个 | 内容创作、文案、视觉设计 |
| 👔 **管理协调类** | 3个 | 项目管理、质量把控、流程优化 |
| 🔍 **研究分析类** | 1个 | 数据调研、竞品分析 |
| 🤝 **用户交互类** | 1个 | 用户沟通、需求分析 |

## 📚 分类 1：文档管理类（4 角色）

### 1. 📚 飞书知识库管理员
**核心定位：** 飞书文档和知识库的专职管理者
**核心能力：**
- ✅ 飞书文档全自动创建与更新
- ✅ 知识库分类体系设计与维护
- ✅ 文档质量审核与标准化
- ✅ 元数据管理与检索优化
**擅长工具：** feishu-automation-suite, 飞书 Bitable API

### 2. 🔧 知识工程师
**核心定位：** 知识体系架构师
**核心能力：**

- ✅ 信息分类与标签体系
- ✅ 知识检索效率优化

### 3. 🔍 研究分析师
**核心定位：** 技术与信息研究员
**核心能力：**
- ✅ 技术趋势调研与分析
- ✅ 竞品信息搜集
- ✅ 研究报告撰写
- ✅ 信息可信度评估

### 4. 📐 信息架构师
**核心定位：** 信息结构设计师
**核心能力：**
- ✅ 文档结构标准化

```yaml
  创建时间: "YYYY-MM-DD"

  - 工具1 (使用场景)
  - 工具2 (使用场景)

  - 输出要求1
  - 输出要求2

协作关系:
  上游: [需要哪些角色的产出]
  下游: [产出给哪些角色]
```

## 🎯 使用指南

### 何时使用此技能
1. 创建新的自动化流程时，选择合适的角色组合
2. 任务分配时，明确各角色的职责边界
3. 优化协作流程时，参考角色协作矩阵
4. 技能开发时，按角色分类组织

### 角色激活原则
- 优先激活高频使用的核心角色
- 低频角色按需启用，避免闲置
- 根据实际使用反馈，动态调整角色能力
