# 🔴 飞书通用坑预警（必看）
**⚠️ 所有飞书操作通用规则：**
- ❌ **-32602 错误码≠通用JSON-RPC参数错误**，在飞书场景下=临时token失效
- ✅ **-32602 唯一修复命令：`lark-cli config bind --source hermes --identity bot-only`**
- ❌ **追加文档内容不要把child_index放JSON payload里**
- ✅ **追加文档内容必须在URL加`?child_index=9999`参数，才能可靠追加到末尾**
- ❌ **多维表格多选字段写入时不能传逗号分隔字符串**（会报 `1254063 MultiSelectFieldConvFail`）
- ✅ **多选字段必须传字符串数组**，如 `["标签1", "标签2"]`
- ❌ **飞书开放API不支持创建日历视图**（会报 `99992402 field validation failed`）
- ✅ **仅支持5种视图：grid(表格)、kanban(看板)、gallery(画廊)、gantt(甘特图)、form(表单)**；日历视图需在飞书界面手动创建

## 🔧 标准工作流程

### 流程 A：新文档自动归档

```
用户调用归档指令
    ↓
① 读取文档内容进行分析
    ├─ 提取标题、正文、创建时间
    ├─ 识别文档类型和主题
    └─ 预估质量等级
    ↓
② 智能分类推荐
    ├─ 匹配 6 大知识库
    ├─ 推荐负责 AI 角色
    ├─ 匹配业务场景
    └─ 推荐标签组合
    ↓
③ 自动补全元数据
    ├─ 生成标准命名（【编码】名称_v版本）
    ├─ 自动计算字数、图片数
    ├─ 设置版本号 v1.0
    ├─ 设置创建/更新时间
    └─ 填写关联信息
    ↓
④ 入库到多维表格
    ├─ 写入文档管理表（23 个字段）
    ├─ 更新 AI 角色库的文档计数
    └─ 触发后续审核流程
    ↓
✅ 返回归档结果 + 知识库链接
```

### 流程 B：文档批量标准化

```
扫描已有文档
    ↓
检查命名规范 → 自动修正
检查元数据完整性 → 自动补全
检查分类准确性 → 重新归类
    ↓
输出标准化报告
```

### 流程 C：知识库月度运营分析

```
拉取整月数据
    ↓
统计：文档总量 + 新增量 + 更新量
分析：5星文档占比 + 元数据完整率
识别：待改进文档 + 空白分类
    ↓
生成运营月报 + 优化建议
```

## ⚠️ 最最最重要的一条 —— 我犯过的愚蠢错误（2026-06-28 深刻教训）

### ❌ 我犯了什么错？

**我竟然想让主人用 Python 脚本在 Linux 服务器上实现飞书多维表格的仪表盘和自动化功能！**

飞书多维表格**本身就自带**这些功能，全部都是在飞书界面里面点几下就配置好的：
- ✅ 飞书多维表格自带 **仪表盘功能**（数字卡片、饼图、条形图、折线图）
- ✅ 飞书多维表格自带 **自动化规则**（记录创建/更新时触发、定时触发）
- ✅ 飞书多维表格自带 **多种视图**（表格、看板、画廊、日历、甘特图）

我竟然想让主人：
1. SSH 登录服务器
2. 运行 `python kb_automation.py dashboard` 看控制台输出
3. 用 Cron 任务代替飞书自带的自动化规则

这完全是本末倒置！**把简单的事情搞复杂了！**

---

### ✅ 正确的认知（必须刻在脑子里）

| 功能 | 应该怎么做 | ❌ 不应该怎么做 |
|------|-----------|----------------|
| **数据统计/仪表盘** | 在飞书多维表格的「仪表盘」功能里配置图表 | 写 Python 脚本在控制台打印数据 |
| **自动化规则** | 在飞书多维表格的「自动化」功能里配置触发器 | 写 Python 脚本 + Cron 定时任务 |
| **数据展示/筛选** | 在飞书多维表格创建不同的「视图」 | 写 Python 脚本过滤数据 |
| **通知提醒** | 用飞书自动化的「发送消息」动作 | 写 Python 脚本调用飞书消息 API |

---

### 🎯 核心原则（永远记住）

**主人用飞书知识库，是为了在飞书里面用，不是为了在 Linux 服务器上用！**

✅ **正确的思路：** 所有功能都应该在飞书界面里完成，主人点几下鼠标就能用

❌ **错误的思路：** 把飞书只当数据库，业务逻辑和展示全用 Python 在服务器上实现

---

### 🚨 触发这个错误的警示信号

当你脑子里出现以下想法时，立即打住！重新思考！

1. ❌ "我写个 Python 脚本来统计..." → ✅ 应该用飞书仪表盘
2. ❌ "我加个 Cron 任务定时检查..." → ✅ 应该用飞书自动化规则
3. ❌ "主人运行这个命令就能看到..." → ✅ 应该在飞书里就能看到

**只要是多维表格能做的事情，绝对不要写 Python 脚本实现！**
