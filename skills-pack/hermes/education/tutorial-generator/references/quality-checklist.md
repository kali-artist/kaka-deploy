# ✅ Quality Checklist - 教程质量 8 项打分自检（v2.1）

> 每次交付前逐项打分，总分 16 分，≥14 分可交付，<12 分必须回滚修复。

## 评分规则

每项 0~2 分：
- **2 分** = 完全达标
- **1 分** = 基本达标，有小瑕疵
- **0 分** = 不达标，必须修

## 🚨 红线项（0 分 = 一票否决）

### 1. 结构清晰（0-2 分）
- [ ] 2分：章节编号 `## 第 N 章：xxx` 统一；每章有 核心→讲解→示例→总结；递进合理
- [ ] 1分：编号统一但某章结构不完整
- [ ] 0分：编号混乱或章节无递进
- **得分：__ / 2**

### 2. 图文并茂·技术图（0-2 分）
- [ ] 2分：至少每 2 章 1 张技术图；图有编号标题；路径正确
- [ ] 1分：图够但缺编号，或某章缺图
- [ ] 0分：整篇无技术图
- **得分：__ / 2**

### 3. 图文并茂·生动插画（0-2 分）⭐ v2.1 新增
- [ ] 2分：开篇+结尾各至少 1 张 mmx 插画；风格统一 flat cartoon；无乱码文字
- [ ] 1分：只有 1 张插画，或风格不统一
- [ ] 0分：无插画（纯 FAQ 类豁免）
- **得分：__ / 2**

## 📋 内容质量项

### 4. 前置知识对齐（0-2 分）
- [ ] 2分：明确列出前置；初学者不要求过多；进阶列出必需项
- [ ] 1分：有前置清单但不够精准
- [ ] 0分：未提及前置知识
- **得分：__ / 2**

### 5. 代码可运行（0-2 分）
- [ ] 2分：所有代码块标语言；关键代码有预期输出；无半段代码
- [ ] 1分：标了语言但缺预期输出
- [ ] 0分：代码块未标语言或不可运行
- **得分：__ / 2**

### 6. 术语一致 + 链接可访问（0-2 分）
- [ ] 2分：术语全篇一致+首现有解释；外链抽查 3 条可访问
- [ ] 1分：术语一致但外链有失效
- [ ] 0分：术语混用或链接大面积失效
- **得分：__ / 2**

### 7. 练习/FAQ 齐全（0-2 分）
- [ ] 2分：T1/T2/T4 每章有练习；T3 末尾≥3 条 FAQ
- [ ] 1分：有练习但不够（某章缺）
- [ ] 0分：无练习无 FAQ
- **得分：__ / 2**

## 🎯 交付元数据

### 8. 模板标注 + 平台可用（0-2 分）
- [ ] 2分：交付消息明确写"T_ 模板"；飞书链接可打开 / MD 图片路径可读
- [ ] 1分：标了模板但平台校验未做
- [ ] 0分：未标模板 or 链接打不开
- **得分：__ / 2**

## 检查方式（自动化优先）

```bash
# 检查图片路径存在（含插画）
grep -oP '!\[.*?\]\(\K[^)]+' tutorial.md | while read img; do
  [ -f "$(dirname tutorial.md)/$img" ] || echo "❌ 图缺失: $img"
done

# 检查代码块是否标注语言
grep -c '^```$' tutorial.md  # 应为 0

# 检查章节编号
grep -c '^## 第 [0-9]' tutorial.md  # 应等于承诺章数

# 检查插画数量（.jpg 文件，illu_ 前缀）
ls images/*illu_*.jpg 2>/dev/null | wc -l  # 开篇+结尾至少 2 张
```
