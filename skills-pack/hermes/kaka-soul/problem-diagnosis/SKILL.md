# 问题诊断与解决技能

## 触发场景
- 工具调用超时或卡住
- 命令执行失败
- 服务异常
- 任何 unexpected error
- 用户明确要求进行问题检查

## 诊断流程

### 第一步：快速自检（30秒内）
```
1. 检查基础服务状态
2. 检查网络连接
3. 检查磁盘空间/内存/CPU
4. 检查相关进程状态
5. 查看最近的错误日志
```

### 第二步：根因分析
根据问题类型，执行针对性诊断：

#### 网络问题
- ping 测试
- DNS 解析检查
- 端口连通性测试
- 代理配置验证

#### 进程/服务问题
- 进程列表检查
- 日志文件分析
- 资源占用检查
- 端口占用检查

#### 文件系统问题
- 磁盘空间检查
- 权限验证
- 文件存在性检查

#### 工具调用问题
- 依赖检查
- 配置验证
- 版本兼容性检查

### 第三步：问题分类与决策

#### 简单问题（自动解决）
符合以下任一条件：
- 明显的配置错误
- 进程僵死需要重启
- 磁盘空间不足但可清理
- 权限问题可快速修复
- 网络临时波动
- 有明确的、低风险的解决方案

**行动：** 直接修复，记录操作，告知用户结果

#### 复杂问题（咨询用户）
符合以下任一条件：
- 需要修改系统核心配置
- 可能影响其他服务
- 解决方案不明确
- 需要安装未知来源的软件
- 可能导致数据丢失
- 修复时间预估超过5分钟
- 需要用户提供额外信息（密码、密钥等）

**行动：**
1. 整理诊断结果
2. 提供2-3个可行方案
3. 说明各方案的利弊和风险
4. 等待用户决策

## 标准诊断命令库

### 系统检查
```bash
# 磁盘空间
df -h

# 内存使用
free -h

# CPU 负载
uptime

# 进程列表
ps aux

# 系统日志
journalctl -n 50 --no-pager
```

### 网络检查
```bash
# 连通性
ping -c 3 8.8.8.8

# DNS
nslookup github.com

# 端口
nc -zv example.com 443

# 代理
echo $HTTP_PROXY $HTTPS_PROXY
```

### 服务检查
```bash
# systemd 服务
systemctl status <service-name>
systemctl --failed

# 端口占用
netstat -tlnp
ss -tlnp
```

## 使用示例

### 场景1：浏览器工具超时
```
1. 检查浏览器进程状态
2. 检查是否有僵死进程
3. 尝试清理进程重启
4. 若仍失败，向用户报告诊断结果
```

### 场景2：磁盘空间不足
```
1. 检查磁盘使用情况
2. 识别可清理的文件（日志、缓存等）
3. 自动清理低风险文件
4. 报告清理结果
```

### 场景3：网络连接失败
```
1. 检查基础网络连通性
2. 检查代理配置
3. 测试多个目标站点
4. 若为配置问题则修复，否则咨询用户
```

## 报告格式

### 简单问题解决报告
```
✅ 问题已自动解决！

问题：[简短描述]
原因：[根因分析]
解决：[执行的操作]
结果：[当前状态]
```

### 复杂问题咨询报告
```
⚠️ 需要您的决策

问题诊断结果：
- 现象：[描述]
- 可能原因：[分析]

可行方案：
1. [方案A] - 利弊：[说明]
2. [方案B] - 利弊：[说明]
3. [方案C] - 利弊：[说明]

请您选择方案，或提供更多指示。
```

## 注意事项
1. 自动修复前，尽量先备份相关配置
2. 涉及数据删除时要格外谨慎
3. 记录所有执行过的操作
4. 即使是简单问题，解决后也要向用户报告
5. 不要尝试猜测密码或绕过安全机制
